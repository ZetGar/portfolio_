# TOALGO 투자 성향 분석 — 멀티 스텝 분석 UX 및 맞춤형 포트폴리오 추천

## 프로젝트 개요

투자 성향 분석 서비스는 사용자의 투자 성향을 분석하고 분석 결과에 따라 맞춤형 투자 포트폴리오를 제공하는 기능입니다.

기존 분석 페이지는 긴 스크롤 기반 구조로 구성되어 있어 사용자가 분석 진행 흐름을 이해하기 어려웠고 UX 개선이 필요한 상황이었습니다.

또한 성향 분석 결과를 기반으로 포트폴리오 추천 결과를 제공하기 위해 분석 데이터 흐름 관리와 결과 API 연동 구조가 필요했습니다.

이를 해결하기 위해 **단계별 분석 UX 구조를 설계하고 분석 데이터 관리, 결과 API 연동, 사용자 흐름을 고려한 인터랙션을 구현했습니다.**

## 문제 상황

### 1. 긴 스크롤 기반 분석 구조

기존 분석 페이지는 모든 질문이 하나의 긴 페이지에 배치되어 있어 사용자가 현재 분석 진행 상황을 파악하기 어려운 문제가 있었습니다.

### 2. 분석 단계 데이터 관리 필요

투자 성향 분석은 1차 성향 분석(5단계) + 2차 포트폴리오 추천 분석(4단계), 총 9단계로 구성되어 있었으며 각 단계의 입력값을 모두 관리해야 결과를 계산할 수 있는 구조였습니다.

### 3. 분석 사용 제한 필요

서비스 정책상 투자 성향 분석은 **하루 최대 3회까지만 가능하도록 제한해야 했습니다.**

### 4. 결과 페이지 이동 구조 문제

분석 결과 페이지에서 추천된 투자 상품으로 이동할 때 **모바일 앱 환경**과 **증권사 웹뷰 환경** 두 가지를 고려해야 하는 구조였습니다.

## 해결 전략

투자 성향 분석 과정을 단계별 UX 구조로 재설계하고, 커스텀 슬라이더 인터랙션과 무한 캐러셀 애니메이션을 직접 구현하여 별도 라이브러리 없이 완성도 높은 UX를 구현했습니다. 또한 분석 횟수 제한 로직과 앱/웹뷰 이동 분기 구조를 적용하여 실서비스 환경에서 안정적으로 동작할 수 있도록 개선했습니다.

## 기술적 구현

### 1. 커스텀 슬라이더 기반 질문 UI 구현

설문 라이브러리(Swiper, Survey.js 등) 없이 **5단계 커스텀 슬라이더 UI를 직접 설계·구현**했습니다.

**구현 내용**

- HTML `input[type="range"]`를 커스터마이징하여 0~4 범위의 **5단계 슬라이더** 구현
- 각 단계별 **동적 색상 라벨** 적용 — 슬라이더 값에 따라 `["#878992", "#8181B0", "#766ED8", "#6658EA", "#5845FE"]` 5색 변경
- 초기 상태에서 **"좌우로 이동해주세요"** 안내 텍스트 표시 → 첫 터치 시 선택값 텍스트로 전환
- `isTouched` 상태로 **첫 인터랙션 감지** — 터치 전까지 다음 버튼 비활성화
- `step` prop 변경 시 슬라이더 값·터치 상태 자동 초기화
- WebView 환경에서 슬라이더 조작 시 페이지 스크롤 방지를 위한 **`touchmove` 이벤트 `preventDefault()` 처리** (passive: false)

> **핵심 포인트**: 외부 라이브러리 없이 range input + CSS + 상태 관리만으로 동적 색상, 안내 텍스트 전환, 터치 감지, WebView 스크롤 방지까지 포함한 완성도 높은 슬라이더 UI를 직접 구현

### 2. 무한 캐러셀 인트로 애니메이션 구현

성향분석 시작 화면에서 5가지 투자 성향 캐릭터가 자동으로 순환하는 **무한 캐러셀**을 직접 구현했습니다.

**구현 내용**

- `translateX` + `transition` 기반 슬라이드 애니메이션
- 마지막 → 첫 번째 슬라이드 전환 시 **`transition: "none"`으로 순간 이동** 처리하여 무한 루프 효과 구현
- `setInterval` 기반 자동 슬라이드 (일반 1초, 리셋 시 350ms로 자연스러운 전환)
- `useRef`로 슬라이더 트랙 DOM 직접 측정하여 슬라이드 폭 동적 계산
- 현재 활성 슬라이드에 `center` 클래스, 나머지에 `scale` 클래스 적용하여 **포커스/축소 효과**

> **핵심 포인트**: Swiper 등 캐러셀 라이브러리 없이 translateX + transition + setInterval 조합으로 무한 루프, 자동 전환, 크기 변화 효과를 모두 구현

### 3. 멀티 스텝 분석 프로세스 구현

총 9단계의 분석 과정을 순차적으로 진행하도록 구현했습니다.

**구현 내용**

- **Indicator 컴포넌트** — `calc(100% / 5 * ${step})`로 프로그레스 바 너비를 동적 계산하여 현재 진행 상태를 시각적으로 표시
- **Question1Template** — 2버튼(예/아니오) 선택 UI (이미지 + 텍스트 카드 방식)
- **QuestionMultiChoice** — 5지선다 슬라이더 UI (단계별 자동 초기화)
- **QuestionCard** — 질문 제목 + 부제목 + children 구조의 재사용 가능한 래퍼 컴포넌트
- `\n` split 기반 **멀티라인 텍스트 렌더링** 처리

### 4. 분석 데이터 서버사이드 관리

**Next.js API Route**(`/api/questions/[id]`)로 설문 데이터를 서버에서 관리하는 구조를 설계했습니다.

**구현 내용**

- 11개 질문 데이터(질문 텍스트, 선택지, 안내 notice)를 **Dynamic Route 파라미터**(`[id]`)로 개별 조회
- `GET /api/questions/1` ~ `GET /api/questions/11` 형태의 RESTful API 설계
- 질문 유형별 구분: `type: "quantec"` (AI 질문) → 향후 사용자 응답(`type: "user"`) 확장 가능
- 404/500 에러 핸들링

> **핵심 포인트**: 설문 데이터를 API Route로 분리하여 질문 콘텐츠 변경 시 프론트엔드 배포 없이 서버 데이터만 수정 가능한 구조

### 5. 성향 결과 페이지 구현

분석 결과를 투자 성향 유형별로 시각적으로 표현하는 결과 페이지를 구현했습니다.

**구현 내용**

- 5단계 투자 성향별 **캐릭터 이미지 · 고유 색상 · 설명 텍스트**를 데이터 배열로 관리하여 `propensity` 인덱스 하나로 전체 UI를 동적 렌더링

| 유형 | 색상 | 설명 |
|---|---|---|
| 안정형 | `#44D18D` | 원금 손실 감내 어려움, 예적금 수준 기대 |
| 안정추구형 | `#48BBFB` | 원금 손실 최소화, 낮지만 안정적 수익 |
| 위험중립형 | `#F1C114` | 원금 손실 대비 적정 수익, 주가지수 수준 |
| 적극투자형 | `#F96A3D` | 높은 수익 위험 인식, 손실 위험 감수 |
| 공격투자형 | `#F53B52` | 손실 위험 적극 수용, 대부분 위험 자산 투자 |

- 사용자 선택 내역(연령, 금융 투자 경험, 이해도, 연간 수입, 투자 목표, 위험 감내 정도) 요약 표시
- **성향 등급 하향 감지** — 이전 분석 대비 등급이 낮아진 경우 리밸런싱 불가 안내
- 안정형(`propensity === 0`) 시 투자 불가 안내 별도 분기
- 동의 체크박스 기반 다음 단계 진행 제어

> **핵심 포인트**: `propensity` 인덱스 하나로 캐릭터 이미지·색상·설명·안내 문구를 모두 동적으로 렌더링하는 데이터 드리븐 UI 설계, 성향 등급 하락 시 서비스 정책 안내까지 반영

### 6. 분석 횟수 제한 로직 구현

서비스 정책에 따라 투자 성향 분석을 **하루 최대 3회까지만 가능하도록 제한하는 로직을 구현했습니다.**

이를 통해 과도한 분석 요청을 방지하고 서비스 정책을 반영했습니다.

### 7. 결과 페이지 이동 분기 처리

분석 결과 페이지에서 추천 상품으로 이동할 때 사용 환경에 따라 이동 방식을 분기 처리했습니다.

**구현 내용**

- 모바일 앱 환경 → 앱 내부 페이지 이동
- 증권사 웹뷰 환경 → 웹뷰 기반 페이지 이동

이를 통해 다양한 서비스 환경에서도 정상적으로 투자 상품 페이지로 이동할 수 있도록 구현했습니다.

## 주요 성과

- 외부 라이브러리 없이 **커스텀 슬라이더 + 무한 캐러셀** 직접 구현
- 멀티 스텝 분석 프로세스 구현 (9단계 순차 진행)
- Indicator 기반 진행 상태 시각화
- 5단계 성향 유형별 데이터 드리븐 결과 페이지 구현
- 하루 3회 분석 제한 로직 구현
- 앱 / 웹뷰 환경 대응 결과 페이지 이동 구조 구현

## 배운 점

이번 프로젝트를 통해 사용자 분석 서비스에서는 UX 흐름 설계와 데이터 흐름 관리가 매우 중요하다는 것을 경험할 수 있었습니다.

특히

- 외부 라이브러리 의존 없이 **커스텀 인터랙션 UI를 직접 설계·구현하는 경험**
- Next.js API Route를 활용한 **설문 데이터의 서버사이드 관리 구조 설계**
- 성향 등급 변화에 따른 **서비스 정책 분기 처리 (안정형 투자 불가, 등급 하락 안내)** 경험
- 앱 / 웹뷰 환경을 고려한 **서비스 흐름 설계**

과정을 통해 프론트엔드 개발자가 사용자 경험뿐 아니라 서비스 정책과 데이터 흐름까지 고려해야 한다는 것을 배우게 되었습니다.
