# 관심종목진단 — 종목 검색 · 선택 · 분석 UI 및 비즈니스 로직 구현

## 프로젝트 개요

PB(Private Banker)가 고객 상담 시 **보유 종목을 진단**하는 관심종목진단 서비스에서, **종목 검색 → 체크리스트 선택 → 비중 입력 → 종목 분석** 앞단 UI 및 비즈니스 로직을 설계·구현한 프로젝트입니다.

종목 분석 이후의 AI 비중 최적화 · 비교 · 리포트 기능은 기존에 구현되어 있었으며, 이 **기존 AI 파이프라인과 연결되는 진입점(종목 선택 → 분석 요청)**을 구현하는 것이 핵심 과제였습니다.

특히 외부 연동 시스템에서 종목코드가 넘어오는 경로와 고객 ID 기반 조회 경로가 공존하는 **멀티 진입 경로** 처리, 종목 선택 시 **최대 종목 수 제한 · 최소보유비중 합계 상한 검증** 등 금융 도메인 특유의 비즈니스 로직을 프론트엔드에서 처리해야 했습니다.

---

## 문제 상황

### 1. 멀티 진입 경로 처리

동일한 종목 조회 화면에 **3가지 서로 다른 진입 경로**가 존재했습니다:
- **고객 ID 기반**: PB가 상담 고객을 선택하여 보유종목 API 호출
- **외부 연동 시스템**: 외부 시스템에서 종목코드/비중 배열이 직접 전달
- **데이터 표출 모드**: 기존 상담 결과를 불러와 조회만 가능

각 경로마다 데이터 소스, 초기 상태, UI 동작이 달라 분기 처리가 복잡했습니다.

### 2. 금융 도메인 비즈니스 로직 검증

종목 선택 시 다음의 비즈니스 규칙을 프론트엔드에서 실시간 검증해야 했습니다:
- 최대 선택 종목 수: **N개 제한**
- 최소보유비중 합계: **상한 초과 불가**
- 개별 최소보유비중: **하한 미만 입력 불가**
- 비중 미입력 시 **자동 기본값 적용**

### 3. 종목명 표시 및 UX 세부 처리

종목명이 셀 너비를 초과하는 경우의 표시 처리, 비중 입력 시 자동 체크 연동 등 사소하지만 사용성에 직결되는 UX 디테일을 챙겨야 했습니다.

---

## 해결 전략

**진입 경로별 분기 로직**을 초기화 단계에서 일원화하고, **실시간 비즈니스 로직 검증**과 **UX 마이크로인터랙션**을 세부적으로 구현하여, 기존 AI 파이프라인으로 정확한 데이터가 전달되도록 설계했습니다.

---

## 기술적 구현

### 1. 종목 조회 — 3가지 진입 경로 통합

동일한 종목 조회 UI에 **3가지 서로 다른 데이터 소스**가 공존하는 구조를 설계했습니다.

**구현 내용**

- **고객 ID 진입**: 고객 ID로 보유종목 API 호출 → 종목 리스트 렌더링
- **외부 연동 시스템 진입**: 전달받은 종목코드 배열에서 `findIndex` + `trim`으로 **코드 매칭** → 비중 자동 매핑
- **데이터 표출 모드**: 기존 상담 결과 데이터로 체크리스트 + 비중 값 **자동 복원**
- 세 경로 모두 동일한 `assetList` 상태로 수렴 → 이후 UI 로직 통일

```
진입 판별 로직:
├── 고객 ID 없음 + 외부 연동 데이터 있음 → 외부 연동 모드
├── 기존 상담 결과 있음 → 데이터 표출 모드
├── 고객 ID 있음 → 고객 보유종목 조회
└── 그 외 → 보유종목 없음 안내 팝업
```

> **핵심 포인트**: 3가지 진입 경로를 초기화 단계에서 분기하고, 이후 UI 로직은 동일한 상태 구조로 통합 — 분기 복잡도를 진입점에 집중하여 이후 코드의 단순성 확보

### 2. 종목 선택 — 실시간 비즈니스 로직 검증

종목 체크박스 선택 시 **금융 도메인의 비즈니스 규칙을 실시간으로 검증**하는 로직을 구현했습니다.

**구현 내용**

- **최대 종목 수 제한**: 체크 시 종목 수 상한 검증 → 초과 시 즉시 알림
- **전체선택 / 개별선택**: 전체선택 시에도 종목 수 제한 동일 적용
- **체크 연동**: 비중 입력 필드 클릭 시 해당 종목 **자동 체크** (체크 안 된 상태에서 비중만 입력하는 실수 방지)
- **체크 해제 시 비중 초기화**: 체크 해제 → 비중 값 `0.00`으로 리셋
- **버튼 상태 자동 제어**: 하나 이상 체크 → '종목 분석' 활성화 / 모두 해제 → 비활성화

> **핵심 포인트**: 금융 서비스에서 잘못된 입력은 실질적 손실로 이어질 수 있어, 비즈니스 규칙을 실시간으로 검증하는 방어적 UI 설계

### 3. 최소보유비중 입력 — 유효성 검증 파이프라인

각 종목의 최소보유비중을 입력하는 과정에서 **다중 유효성 검증**을 실시간으로 수행하는 구조를 구현했습니다.

**구현 내용**

- **숫자만 허용**: 정규식으로 비숫자 입력 차단
- **개별 상한 검증**: 단일 종목 비중이 합계 상한 초과 시 입력 거부 + 알림
- **합계 검증**: 전체 종목 비중 합계가 상한 초과 시 해당 입력 값을 0으로 리셋 + 알림
- **포커스 아웃 자동 보정**: blur 시 0%이면 기본값(기존 보유비중과 기본값 중 작은 값)으로 자동 복원
- **소수점 포맷팅**: blur 시 `toFixed(2)`로 소수점 2자리 자동 정리

```
비중 입력 검증 파이프라인:
입력 → 숫자 검증 → 개별 상한 검증 → 합계 검증 → 상태 업데이트
포커스 아웃 → 0% 체크 → 기본값 복원 → 소수점 포맷팅
```

> **핵심 포인트**: 입력 → 검증 → 보정 → 포맷팅까지 4단계 파이프라인으로 잘못된 비중 입력을 원천 차단

### 4. UX 마이크로인터랙션

사소하지만 사용성에 직결되는 **세부 UX 처리**를 구현했습니다.

**구현 내용**

- **종목명 말줄임 + Tooltip 자동 표시**: `scrollWidth > clientWidth` 비교로 텍스트 오버플로우 감지 → 오버플로우 시 `...` 표시 + Tooltip 아이콘 자동 노출 → 클릭 시 5초 후 자동 닫힘
- **비중 입력 클릭 → 자동 체크**: 비중 입력 필드 클릭 시 해당 종목이 자동으로 체크 — "체크 안 하고 비중만 입력하는" 실수 방지
- **비중 입력 필드 클릭 → 값 초기화**: 기존 값이 있어도 클릭 시 빈 값으로 초기화하여 바로 입력 가능
- **포트폴리오 생성 후 그레이스케일 처리**: `filter: grayscale(0.7)`로 분석 완료 영역 시각적 비활성화
- **재분석하기 버튼**: 분석 완료 후 종목 재선택이 필요한 경우 체크리스트 · 비중 · 버튼 상태 전체 초기화

> **핵심 포인트**: 금융 전문가(PB)가 실수 없이 빠르게 조작할 수 있도록 입력 시나리오를 예측하고 자동 보정하는 방어적 UX

### 5. 기존 AI 파이프라인 연동

종목 선택 · 비중 입력이 완료되면 **기존 AI 비중 최적화 시스템으로 데이터를 전달**하는 연동 구조를 구현했습니다.

**구현 내용**

- 체크된 종목의 `shortStockCode` + 입력된 `minimumHoldingWeight`를 정리하여 분석 API 요청 데이터 생성
- 비중 미입력 종목은 기본 하한값 자동 적용 후 전송
- 전송 전 최종 합계 검증 → 통과 시 로딩 스피너 표시 + API 호출
- 분석 결과를 받아 기존 AI 코칭 결과 화면으로 데이터 전달

> **핵심 포인트**: 앞단에서 검증된 깨끗한 데이터를 기존 AI 파이프라인에 전달 — 입력 검증을 진입점에서 완결하여 후단 시스템의 오류 가능성 최소화

특히
