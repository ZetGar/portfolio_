# 금융 투자 플랫폼 (Part 1) — 전략마켓 대규모 데이터 비교 UX 개선 및 서비스 고도화

## 프로젝트 개요

증권사 고객에게 제공되는 투자 전략 비교 서비스(전략마켓)의 프론트엔드를 전면 개선한 프로젝트입니다.

기존 페이지는 Tailwind와 MUI가 혼용된 스타일 구조로 유지보수가 어려웠으며, Mobile/PC 코드가 분리되어 동일 기능 코드가 중복되는 문제가 있었습니다. 또한 법적 요구사항에 따라 모든 상품 데이터를 공개해야 하면서 대규모 데이터 테이블의 가독성과 성능 문제가 발생했습니다.

이를 해결하기 위해 **스타일 구조 리팩토링, Sticky Header / Column 기반 테이블 UX 개선, Highcharts 데이터 시각화, 비동기 데이터 로딩 구조를 적용하여 전략마켓 서비스를 전면 개선했습니다.**

---

## 문제 상황

### 1. 스타일 구조 혼재 및 코드 중복

Tailwind와 MUI가 혼용되면서 동일 UI가 서로 다른 방식으로 구현되어 스타일 관리가 어려웠습니다. 또한 Mobile/PC 코드가 분리되어 동일 기능의 코드가 중복되는 구조였습니다.

### 2. 대규모 데이터 테이블 가독성 문제

규제 대응으로 모든 데이터를 공개해야 하면서 테이블이 viewport를 초과했고, 가로 스크롤 시 전략명이 사라져 어떤 전략의 데이터인지 확인이 어려웠습니다.

### 3. 차트 데이터 로딩 성능 문제

차트 데이터의 양이 많아 모든 데이터를 한 번에 로딩할 경우 초기 페이지 렌더링이 지연되는 문제가 있었습니다.

### 4. 차트 및 UI 디자인 불일치

전략마켓과 내 자산 페이지에서 사용하는 차트가 각각 개별 구현되어 디자인과 데이터 처리 방식이 서로 달랐습니다.

### 5. 브라우저 업데이트 대응 필요

Chrome 업데이트 이후 Sticky 동작 방식이 변경되면서 일부 테이블에서 컬럼이 어긋나거나 스크롤이 비정상 동작하는 문제가 발생했습니다.

---

## 해결 전략

스타일 구조를 CSS Module 기반으로 재구성하고, Sticky Header / Column 이중 구조와 Highcharts 데이터 시각화, 비동기 데이터 로딩을 적용하여 대규모 금융 데이터의 비교 가독성과 사용자 경험을 전면 개선했습니다.

---

## 기술적 구현

### 1. CSS 구조 리팩토링 (Tailwind + MUI → CSS Module)

Tailwind와 MUI가 혼재된 스타일 구조를 **CSS Module 기반으로 전면 재구성**했습니다.

**구현 내용**

- 기존 Tailwind 기반 스타일을 CSS Module 구조로 변환
- Mobile/PC 분리 코드를 통합하여 중복 제거
- 상품 유형별(일반 상품 / 세제혜택 상품) 표시 로직을 정리하여 렌더링 분기 단순화

> **핵심 포인트**: 스타일 충돌 문제 해결과 함께, 상품 유형별 표시 조건(연말정산 대상 상품 등)의 UI 로직을 정리하여 유지보수성 확보

### 2. Sticky Header / Column 이중 구조 테이블 구현

가로·세로 스크롤에서 핵심 정보가 항상 보이도록 **이중 Sticky 구조**를 직접 설계·구현했습니다.

**구현 내용**

- `sticky_y` — 세로 스크롤 시 **헤더 행 고정** (`position: sticky; top: 0; z-index: 5`)
- `sticky_x` — 가로 스크롤 시 **전략명 컬럼 고정** (`position: sticky; left: 0; z-index: 10`)
- 헤더와 전략명이 교차하는 셀에 `sticky_y + sticky_x` **동시 적용** → 스크롤 방향과 무관하게 항상 고정
- `z-index` 레이어링으로 헤더(5) → 바디 전략명(1) → 교차 셀(10) 겹침 순서 제어
- `::before` 가상 요소로 sticky 셀의 **배경색·보더 깨짐 방지** 처리
- **좌우 스크롤 버튼** — `scrollBy({left: ±300, behavior: 'smooth'})` + `scrollLeft` 위치 감지로 prev/next 버튼 동적 표시/숨김
- **전략 카테고리별 색상 배지** 자동 매핑 (국내주식: 초록, 해외주식: 파랑, 연금: 주황, ETF: 보라)

> **핵심 포인트**: CSS `position: sticky`만으로는 가로+세로 동시 고정 시 배경 겹침, z-index 충돌 문제가 발생 — `::before` 가상 요소 + z-index 3단계 레이어링 + 좌우 스크롤 버튼으로 OS별 환경에서 안정적으로 동작하도록 해결

### 3. Chrome 업데이트 Sticky 동작 변경 대응

Chrome 업데이트 이후 Sticky 렌더링 방식이 변경되면서 일부 테이블 레이아웃이 깨지는 문제가 발생했습니다.

**구현 내용**

- 브라우저 동작 방식 변화를 분석하고 Sticky 적용 CSS 구조 재정비
- 변경된 렌더링 규칙에 대응하도록 CSS 수정

> **핵심 포인트**: 브라우저 업데이트에 따른 CSS 렌더링 변화를 추적하고 대응한 실무 경험

### 4. Highcharts 기반 데이터 시각화

테이블 각 행에 **Highcharts 미니 Area 차트를 인라인으로 렌더링**하고, 상세 페이지에서는 전략 수익률 라인 차트를 구현했습니다.

**구현 내용**

- 미니 차트 옵션 유틸리티로 차트 설정 표준화
- tooltip 표시 데이터 포맷 수정 및 hover 시 데이터 강조 방식 개선
- 데이터 없는 전략에 대한 fallback("No Data") 처리
- **수익률 색상 자동 매핑**: 양수(빨강)/음수(파랑)/데이터 미존재(-) 자동 표시

> **핵심 포인트**: 테이블 행 내부에 차트를 인라인으로 렌더링하여, 텍스트 수치와 시각적 추세를 동시에 비교할 수 있는 금융 서비스 특화 UX

### 5. 차트 · 미니차트 공통 컴포넌트화

전략마켓과 내 자산 페이지에서 각각 개별 구현되어 있던 차트를 **재사용 가능한 공통 컴포넌트로 통합**했습니다.

**구현 내용**

- 차트 및 미니 차트 UI를 공통 컴포넌트로 추출
- 데이터 누락 문제 수정
- 차트 디자인 및 데이터 구조 통일

> **핵심 포인트**: 여러 페이지에 분산된 동일 유형 차트를 하나의 공통 컴포넌트로 통합하여 코드 중복 제거 + 디자인 일관성 확보

### 6. 비동기 데이터 로딩 및 Skeleton UI 적용

대량 차트 데이터의 초기 로딩 지연 문제를 해결하기 위해 **텍스트와 차트 데이터를 분리하여 비동기 로딩**하는 구조를 적용했습니다.

**구현 내용**

- 텍스트 및 테이블 데이터와 차트 데이터를 **별도 API로 분리 호출**
- 텍스트 데이터를 먼저 렌더링하여 초기 페이지 표시 속도 개선
- 차트 영역은 **Skeleton UI**로 표시 후 데이터 로딩 완료 시 렌더링

> **핵심 포인트**: 사용자 체감 로딩 시간을 단축하기 위해 "데이터 도착 순서대로 렌더링"하는 점진적 렌더링 전략 적용

### 7. 전략 상세 페이지 구현

전략별 기본정보 · 자산구성 · 운용계획 · 수익률 차트를 **좌우 분할 레이아웃**으로 구현했습니다.

**구현 내용**

- **좌측**: DropDown 아코디언으로 기본정보 / 자산구성 / 운용계획 / 운용알고리즘 토글
- **우측**: 기간수익률 테이블 + 위험지표 테이블 + 전략 수익률 라인 차트
- `format` 분기(currency/percent/performance)로 금액·비율·성과 **자동 포맷팅**
- 설정 배열 기반 **데이터 드리븐** 표시 항목 관리
- 위험지표 안내 모달 + 종목 상세 모달 연동

### 8. 차트 데이터 기간별 캐싱 전략

수익률 차트의 기간 탭(6개월/1년/2년/3년/누적) 전환 시 **이미 불러온 데이터를 캐싱하여 불필요한 API 재호출을 방지**하는 구조를 구현했습니다.

**구현 내용**

- 기간별 차트 데이터를 상태에 캐싱
- 같은 탭 재클릭 시 캐시에서 즉시 차트 렌더링 → 네트워크 요청 없이 즉각 응답
- 차트 로딩 중에는 **Skeleton UI**로 로딩 상태 표시

> **핵심 포인트**: 사용자가 기간 탭을 반복 전환하는 패턴에 대응 — 캐시 히트 시 네트워크 비용 0으로 UX 향상

### 9. 공시기간 기반 동적 탭 생성 및 벤치마크 비교

전략마다 공시기간이 다르기 때문에 **공시기간에 따라 차트 기간 탭을 동적으로 생성**하고, 전략 수익률을 **벤치마크 지수와 비교**하여 차트에 함께 렌더링했습니다.

**구현 내용**

- 공시기간 1~2년이면 `['6개월', '1년', '누적']`, 3년 이상이면 `['6개월', '1년', '3년', '누적']` 탭 자동 생성
- 벤치마크 수익률을 전략 수익률과 동일 차트에 오버레이 렌더링
- 투자 성향별(안정/중립/위험추구) 색상 자동 매핑

> **핵심 포인트**: 데이터 조건에 따라 UI가 자동 적응하는 데이터 드리븐 구조 + 벤치마크 비교로 금융 도메인에서 요구하는 상대 수익률 시각화 구현

### 10. 설명 텍스트 Key 기반 서버 관리

화면 내 다양한 지표 설명을 효율적으로 관리하기 위해 **설명 텍스트를 서버에서 Key 기반으로 관리**하는 구조를 구축했습니다.

**구현 내용**

- 설명 텍스트를 서버에 저장하고 고유 Key 부여
- UI에서 도움말 아이콘 클릭 시 Key 기반으로 설명 데이터 조회 → 팝업 표시
- 동일 설명을 여러 화면에서 재사용 가능

> **핵심 포인트**: 코드 수정 없이 설명 문구를 서버에서 관리 가능한 구조 — 금융 서비스에서 자주 변경되는 법적 고지 문구 대응에 효과적

### 11. iOS / Android / WebView UI 크로스 체크

모바일 앱과 웹뷰 환경에서 UI 동작 차이가 발생하지 않도록 **모바일 팀과 협업하여 크로스 플랫폼 UI 검증**을 수행했습니다.

**구현 내용**

- iOS / Android / WebView 환경에서 UI 테스트 수행
- 상단 계좌 클릭 시 UI 밀림 문제 해결 (`position` 속성 기반 레이아웃 구조 개선)
- 환경별 UI 차이가 발생하지 않도록 구조 수정

## 배운 점

이번 프로젝트를 통해 대규모 금융 데이터 환경에서의 UI 설계와 사용자 경험 개선의 중요성을 경험할 수 있었습니다.

특히

- CSS `position: sticky`의 **가로+세로 동시 적용 시 렌더링 이슈와 브라우저 업데이트 대응 경험**
- Highcharts를 **테이블 셀 내부에 인라인으로 렌더링**하고 공통 컴포넌트로 통합하는 설계 경험
- **비동기 데이터 로딩 + Skeleton UI + 기간별 캐싱**을 조합한 성능 최적화 전략
- 모바일 팀과 협업하여 **iOS / Android / WebView 간 UI 일관성을 검증하고 조정한 경험**

과정을 통해 프론트엔드 개발자가 데이터 비교 UX에서 **스크롤, 레이어링, 성능, 크로스 플랫폼까지 고려해야 한다는 점**을 배우게 되었습니다.
